# Story 1c.1: Create shared accessor module for v2 schema

**As a** developer,
**I want** a shared accessor module that both Editor and Suite can import,
**So that** all character data access goes through one consistent API.

## Acceptance Criteria

**Given** the integration plan defines accessor functions for v2 schema
**When** the module is created
**Then** `js/shared/accessors.js` exports all accessor functions from the integration plan
**And** it includes attribute, skill, merit, power, and derived stat accessors
**And** the Editor's existing `js/data/accessors.js` is replaced by or merged with the shared module
**And** all Editor modules that import from `js/data/accessors.js` are updated
