# Story 2.7: Automated Roll Workflows and Session Log

**Status:** Draft

## Story

**As a** Storyteller,
**I want** to run automated contested, social, and territory bidding roll workflows during a session and have all results stored in a session log keyed to date and character,
**so that** roll history is preserved for reference and I can run live roll resolutions without manual dice.

## Acceptance Criteria

1. The Roll tab in the main SPA supports a "Territory Bid" workflow: ST selects two characters (or a character vs. an NPC pool), the system constructs both pools and runs a contested roll, and displays the winner with margin.
2. The Roll tab supports a "Social Manoeuvre" contested workflow: attacker pool (Manipulation + Socialise or similar) vs. defender's Composure + BP resistance.
3. The Roll tab supports a "Resistance Check" contested workflow: attack pool vs. Stamina + BP (physical) or Resolve + Composure (mental) resistance pools.
4. All automated roll results (contested or standard) are written to a session log stored under `localStorage` key `tm_session_log`.
5. Session log entries contain: `{ session_date, character_name, roll_type, pool_expression, dice_string, successes, outcome_note, timestamp }`.
6. The ST can view the session log for the current session (current date) in a new "Log" tab or as a collapsible panel.
7. Log entries are associated with the characters involved; clicking a log entry navigates to that character's sheet (or highlights them in the Roll tab).
8. The session log persists across page reloads and is not cleared by normal app operations.
9. ST can export the current session log as a JSON file.
