# Story 1a.3: Extract reference data from tm_editor.html into JSON files

**As a** developer,
**I want** the large inline data blocks extracted from the editor into separate JSON files,
**So that** data is decoupled from logic and can be loaded, updated, or replaced independently.

## Acceptance Criteria

**Given** `tm_editor.html` contains inline data blocks: `ICONS` (~117K), `MAN_DB` (~44K), `MERITS_DB` (~26K), `DEVOTIONS_DB` (~11K), `CHARS_DATA` (~51K)
**When** the extraction is complete
**Then** each block is a separate JSON file in `data/`: `icons.json`, `man_db.json`, `merits_db.json`, `devotions_db.json`, `chars_test.json`
**And** a loader module (`js/data/loader.js`) fetches these files and exposes them to the app
**And** the editor functions identically - all merit lookups, devotion references, manoeuvre displays, and icon rendering work as before
**And** the inline data blocks are removed from the HTML file
**And** `CHARS_DATA` becomes `data/chars_test.json` containing the 6 fake test characters
