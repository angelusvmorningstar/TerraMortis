# Story 1a.5: Extract JavaScript from index.html (ST Suite) into ES modules

**As a** developer,
**I want** all JavaScript from the ST Suite extracted into modular `.js` files,
**So that** Suite logic is maintainable and modular.

## Acceptance Criteria

**Given** `public/index.html` has two `<script>` blocks: main (lines 797-2621, ~1,824 lines) and Territory React component (lines 2624-2872, ~248 lines)
**When** the extraction is complete
**Then** the `<script>` blocks are replaced with `<script type="module" src="js/main.js">`
**And** JS is split into modules:

- `js/suite/roll.js` - Roll tab UI, pool construction, `getPool()`, `updResist()`, modifier chips
- `js/suite/sheet.js` - Suite sheet view (read-only, mobile-friendly)
- `js/suite/territory.js` - Territory tab (keep as-is for now, React rewrite is sub-phase 1d)
- `js/suite/tracker.js` - Session tracker tab
- `js/shared/dice.js` - core dice rolling engine (10-again, 9-again, 8-again, rote, chance)
- `js/shared/pools.js` - `getPool()` pool string parser
- `js/shared/resist.js` - resistance check calculation
- `js/shared/influence.js` - influence total calculation across 16 spheres

**And** all modules use ES module `import`/`export` with `.js` extensions
**And** no single JS file exceeds 500 lines
**And** the Suite functions identically: Roll tab, Sheet tab, Territory tab, Tracker tab
**And** the React CDN scripts remain for now (Territory tab rewrite is 1d)
