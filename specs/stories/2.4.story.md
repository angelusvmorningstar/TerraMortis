# Story 2.4: Print Character Sheet

**Status:** Draft

## Story

**As a** Storyteller,
**I want** to generate a print-formatted character sheet for any loaded character,
**so that** I can hand physical or PDF copies to players without manually transcribing data.

## Acceptance Criteria

1. A "Print Sheet" button is available on the character sheet view; clicking it opens the print layout in a new browser tab (or triggers `window.print()` from a dedicated print view).
2. The print sheet renders: name, concept, clan, covenant, mask/dirge, apparent age, pronouns, blood potency, humanity, all attributes with dots, all skills with specialisations, all disciplines and discipline dot totals, all powers (discipline powers, rites, devotions, pacts) with name and effect, all merits grouped by category, touchstones with humanity association, banes, aspirations, and all derived stats (size, speed, defence, armour, health, willpower max, vitae max, vitae per turn).
3. Derived stats are calculated at render time via `js/data/derived.js` -- they are never read from stored values.
4. The print layout uses a print-optimised CSS: white background, black text, no dark theme colours, no decorative background images, font sizes legible at A4.
5. The print CSS is in a dedicated `public/css/print.css` and is applied only within the print view (does not affect the main app).
6. Print view renders correctly on all 6 test characters with no console errors.
7. The print sheet does not expose `merit_creation`, XP cost breakdowns, or internal tracking fields -- these are for ST use only.

- [ ] Task 4: Touchstones, banes, aspirations render (AC: 2)
  - [ ] Touchstones: grouped by humanity level, name + description
  - [ ] Banes: name + effect
  - [ ] Aspirations: list

### Derived Stats Formulae
| Stat | Formula |
|---|---|
| Size | 5 (default), modified by Merits |
| Speed | Strength.dots + Dexterity.dots + 5 |
| Defence | min(Wits, Dexterity).dots + Athletics.dots |
| Health | Stamina.dots + Size |
| Willpower max | Resolve.dots + Composure.dots |
| Vitae max | Blood Potency table lookup |
| Vitae per turn | Blood Potency table lookup |
[ref: data-models.md, Derived Stats section]

### Accessors to Use
```js
attrTotal(c, 'Strength')     // for derived stats
skTotal(c, 'Athletics')      // for defence
meritsByCategory(c, cat)     // for grouped merit display
powersByCategory(c, cat)     // for powers list
discDots(c, name)             // discipline dot totals
```
[ref: data-models.md, Accessor Layer]

### Display Conventions
- Dots: `'●'.repeat(n) + '○'.repeat(max - n)` using U+25CF / U+25CB [ref: coding-standards.md]
- Fonts: Cinzel Decorative for headings, Lora for body -- these are Google Fonts CDN; include link in print view [ref: coding-standards.md, Typography]
- British English: "Defence", "Armour", "Willpower" [ref: coding-standards.md]

### CSS Print Standard
- `public/css/print.css` is the only file that may use `@media print` or override dark theme tokens [ref: unified-project-structure.md, `css/` module responsibilities]
- No hardcoded hex in `print.css` -- exception: white (#fff) and black (#000) are acceptable for print where no design token exists [ref: coding-standards.md, Design Tokens -- note the spirit: tokens exist for the dark theme; print is explicitly different]
