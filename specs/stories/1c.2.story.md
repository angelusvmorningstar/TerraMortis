# Story 1c.2: Migrate Suite to read v2 character data

**As a** developer,
**I want** the Suite to read v2 format character data via shared accessors,
**So that** both apps use the same data format and a single data source.

## Acceptance Criteria

**Given** the shared accessor module exists
**When** the Suite migration is complete
**Then** Suite's CHARS_DATA is replaced with v2 format test characters
**And** Suite's loadChars() reads v2 from localStorage (same key as Editor)
**And** all ~44 direct data access points in Suite JS are refactored to use shared accessors
**And** getPool(), getResistTokenVal(), renderSheet(), and tracker functions all work with v2 data
**And** the Suite functions identically: Roll, Sheet, Tracker, Territory tabs all work
