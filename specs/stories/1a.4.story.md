# Story 1a.4: Extract JavaScript from tm_editor.html into ES modules

**As a** developer,
**I want** all JavaScript from the editor extracted into modular `.js` files,
**So that** each module has a clear responsibility, is under 500 lines, and can be worked on independently.

## Acceptance Criteria

**Given** `tm_editor.html` has a `<script>` block (lines 525-3119, ~2,594 lines)
**When** the extraction is complete
**Then** the `<script>` block is replaced with `<script type="module" src="js/main.js">`
**And** JS is split into modules following the target structure:

- `js/editor/list.js` - `renderList()`, character card grid, filtering
- `js/editor/sheet.js` - `renderSheet()` orchestrator and the 9 `shRender*` section functions
- `js/editor/edit.js` - edit mode, `shEdit()`, `markDirty()`, form handlers
- `js/editor/merits.js` - merit add/edit/remove, `buildMeritOptions()`, prerequisite validation
- `js/editor/mci.js` - MCI benefit grants, `applyDerivedMerits()`, suspend/activate
- `js/editor/domain.js` - domain merit sharing maths (`domMeritContrib`, `domMeritShareable`, `domMeritTotal`)
- `js/editor/xp.js` - XP log, `xpToDots()`, creation point allocation, `meritBdRow()`
- `js/editor/export.js` - `saveChars()`, `charsForSave()`, `syncToSuite()`
- `js/data/derived.js` - derived stat calculations (size, speed, defence, health, WP max, vitae max)
- `js/main.js` - app init, navigation, shared state

**And** all modules use ES module `import`/`export` with `.js` extensions
**And** all modules use `const` by default, `let` where needed, never `var`
**And** no single JS file exceeds 500 lines
**And** the editor functions identically: list view, sheet view, edit mode, save, all merit operations
