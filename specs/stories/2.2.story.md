# Story 2.2: MCI Benefit Grants -- Complete Wiring

**Status:** Draft

## Story

**As a** Storyteller,
**I want** all Mystery Cult Initiation benefit grants on test characters to be wired as structured `benefit_grants` with correct prerequisite checking and suspend/activate behaviour,
**so that** granted merits appear accurately on sheets and feed correctly into influence totals.

## Acceptance Criteria

1. All 5 test characters with MCI merits have structured `benefit_grants` arrays (replacing static `granted_by` annotations) in `chars_test.json`; Whisper is already done and must not regress.
2. `applyDerivedMerits(c)` correctly injects each grant as a `derived: true` merit at render time, using the character's active MCI dot level as the ceiling.
3. Grants that fail prerequisite checks render in view mode with a visible "Invalid" tag and strikethrough styling; passing grants render with a gold "Grant" tag.
4. Setting an MCI merit's `active` field to `false` removes all its derived grants from the sheet view and from influence calculations immediately on re-render.
5. The influence total (`influenceTotal(c)`) does not count any merit derived from a suspended or prereq-failed MCI grant.
6. All 6 test characters load and render without console errors after this change.

### MCI Data Structure
```json
{
  "category": "standing",
  "name": "Mystery Cult Initiation",
  "rating": 5,
  "cult_name": "The Hollow Archive",
  "active": true,
  "benefits": ["", "", "", "", ""],
  "benefit_grants": [
    { "category": "general", "name": "Eidetic Memory", "rating": 1 },
    { "category": "general", "name": "Indomitable",    "rating": 2 },
    { "category": "general", "name": "Peacemaker",     "rating": 3 },
    { "category": "general", "name": "Closed Book",    "rating": 3 },
    { "category": "general", "name": "Table Turner",   "rating": 1 }
  ]
}
```
The `benefit_grants` array is indexed 0-4 (dot levels 1-5). Grants at index N are only injected when `merit.rating >= N+1`.
[ref: HANDOVER_v4.md, Section 4, Data Structure]
